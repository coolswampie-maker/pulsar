# ПУЛЬСАР — демонстрационная витрина и бронирование

Статический демо-сайт платформы управления лабораторной инфраструктурой ИНТЦ МГУ
«Воробьёвы горы». Витрина ресурсов (помещения · оборудование · специалисты ·
услуги под ключ), корзина бронирования с реальной логикой и демо-кабинет оператора.

Работает **без сборки и без сервера** — открывается двойным кликом по `index.html`
и разворачивается на GitHub Pages.

## Структура

```
site/
├── index.html            каркас: навигация, футер, подключение скриптов
├── css/styles.css        дизайн-система «академический институциональный»
├── data/
│   ├── resources.js      КАТАЛОГ (source of truth): помещения/оборудование/…
│   └── images.js         манифест изображений (реальные фото ↔ плейсхолдеры)
├── js/
│   ├── images.js         реальное фото из манифеста или брендовый SVG-плейсхолдер
│   ├── api.js            слой данных (граница для будущего backend), даты, занятость
│   ├── store.js          корзина: единицы брони, оператор, конфликты, смета, заявки
│   └── app.js            hash-роутер и все экраны
└── assets/img/           реальные фото (building.jpg, lab-generic.jpg, …)
```

## Что реализовано

- **4 типа ресурсов** с разными единицами брони: помещения (смена), оборудование
  (сутки/час), специалисты (час), услуги под ключ (образец).
- **Авто-подтягивание оператора**: приборы «с оператором» добавляют слот специалиста
  на то же время; он удаляется вместе с прибором.
- **Проверка конфликтов**: занятые слоты (демо-расписание + подтверждённые заявки)
  нельзя забронировать; в панели брони они подсвечены.
- **Смета** с льготой резидента ИНТЦ −25 % (217-ФЗ).
- **Гостевая заявка** → сохраняется локально; **демо-кабинет оператора** (`#/admin`):
  заявки со статусами, календарь загрузки, CRUD каталога.

## Как заменить плейсхолдеры на реальные фото

1. Положите файл в `assets/img/`, напр. `assets/img/sem.jpg`.
2. В `data/images.js` впишите путь по ключу ресурса (поле `img` в `resources.js`),
   напр. `'eq-sem':'assets/img/sem.jpg'`.
   Пустая строка `''` → рисуется аккуратный брендовый плейсхолдер.

## Как добавить/поправить позиции каталога

Отредактируйте `data/resources.js` (постоянно) — или через демо-CRUD в кабинете
оператора (сохраняется в браузере, «Сбросить каталог» возвращает к исходному).

## Публикация на GitHub Pages

1. Создайте репозиторий и запушьте **содержимое папки `site/`** в корень ветки
   (или положите `site/` и укажите её как источник Pages).
2. Settings → Pages → Deploy from branch → выберите ветку и папку.
3. Готово. Файл `.nojekyll` уже включён (чтобы Pages отдавал всё как есть).

## Переход на реальный backend (следующий этап)

Вся работа с данными изолирована в `js/api.js` и `js/store.js`. Достаточно заменить
чтение `window.PULSAR.baseResources` / `localStorage` на `fetch()` к REST API —
интерфейс (`app.js`) менять не потребуется. Модель данных описана в шапке
`data/resources.js`.
